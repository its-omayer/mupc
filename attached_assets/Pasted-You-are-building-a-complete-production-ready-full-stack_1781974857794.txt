RULE 1: ALL image uploads go through your own Next.js API route.

RULE 2: The upload flow is:

    api_key: process.env.CLOUDINARY_API_KEY!,
    api_secret: process.env.CLOUDINARY_API_SECRET!,
    secure: true,
  })

  export default cloudinary

Import this ONLY in server-side files (API routes, server actions).
NEVER import cloudinary in any file that has "use client".

RULE 4: Upload API route — /api/upload/route.ts:

  import { NextRequest, NextResponse } from 'next/server'
  import { getServerSession } from 'next-auth/next'
  import { authOptions } from '@/lib/auth'
  import cloudinary from '@/lib/cloudinary'

  export async function POST(request: NextRequest) {
    try {
      const session = await getServerSession(authOptions)
      if (!session?.user) {
        return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
      }

      const formData = await request.formData()
      const file = formData.get('file') as File | null
      const folder = (formData.get('folder') as string) || 'mupc/uploads'

      if (!file) {
        return NextResponse.json({ error: 'No file provided' }, { status: 400 })
      }

      if (!file.type.startsWith('image/')) {
        return NextResponse.json({ error: 'File must be an image' }, { status: 400 })
      }

      if (file.size > 10 * 1024 * 1024) {
        return NextResponse.json({ error: 'File too large. Max 10MB.' }, { status: 400 })
      }

      const bytes = await file.arrayBuffer()
      const buffer = Buffer.from(bytes)
      const base64 = buffer.toString('base64')
      const dataUri = `data:${file.type};base64,${base64}`

      const result = await cloudinary.uploader.upload(dataUri, {
        folder: folder,
        resource_type: 'image',
        transformation: [
          { quality: 'auto', fetch_format: 'auto' },
          { width: 2000, height: 2000, crop: 'limit' },
        ],
      })

      return NextResponse.json({
        cloudinaryUrl: result.secure_url,
        cloudinaryPublicId: result.public_id,
      })
    } catch (error) {
      console.error('[API /upload]', error)
      return NextResponse.json({ error: 'Upload failed' }, { status: 500 })
    }
  }

━━━ HOW THE AUTH FLOW WORKS ━━━

Both pages use the same NextAuth CredentialsProvider and the same
signIn('credentials', { email, password, redirect: false }) call.
The role check after signIn() is what separates them.

In src/lib/auth.ts, the authorize() function:
1. Trims and lowercases the email
2. Finds the user in MongoDB by email
3. If user not found: return null
4. If user.isActive === false: throw new Error('ACCOUNT_DISABLED')
5. bcrypt.compare(password, user.password)
6. If wrong password: return null
7. Return { id: user._id.toString(), email, name, role, profilePhotoUrl }

In the callbacks:
jwt: add token.id, token.role, token.profilePhotoUrl from user object
session: add session.user.id, session.user.role, session.user.profilePhotoUrl

━━━ MIDDLEWARE: middleware.ts ━━━

import { getToken } from 'next-auth/jwt'
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  const token = await getToken({ req: request, secret: process.env.NEXTAUTH_SECRET })
  const { pathname } = request.nextUrl

  // Admin routes — require admin role
  if (pathname.startsWith('/bash-mubc-bash')) {
    // Allow the login page itself
    if (pathname === '/bash-mubc-bash/login') {
      // If already logged in as admin, redirect to dashboard
      if (token?.role === 'admin') {
        return NextResponse.redirect(new URL('/bash-mubc-bash', request.url))
      }
      return NextResponse.next()
    }
    // All other admin routes require admin role
    if (!token || token.role !== 'admin') {
      return NextResponse.redirect(new URL('/bash-mubc-bash/login', request.url))
    }
    return NextResponse.next()
  }

  // Member dashboard — require any session
  if (pathname.startsWith('/dashboard')) {
    if (!token) {
      return NextResponse.redirect(
        new URL(`/signin?callbackUrl=${encodeURIComponent(pathname)}`, request.url)
      )
    }
    return NextResponse.next()
  }

  // Admin API routes — require admin role
  if (pathname.startsWith('/api/admin')) {
    if (!token || token.role !== 'admin') {
      return NextResponse.json({ error: 'Forbidden' }, { status: 403 })
    }
    return NextResponse.next()
  }

  // Dashboard API routes — require any session
  if (pathname.startsWith('/api/dashboard')) {
    if (!token) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
    }
    return NextResponse.next()
  }

  return NextResponse.next()
}

export const config = {
  matcher: [
    '/bash-mubc-bash/:path*',
    '/dashboard/:path*',
    '/api/admin/:path*',
    '/api/dashboard/:path*',
  ],
}

MONGODB — CRITICAL RULES

RULE 1: MongoDB connection — src/lib/mongodb.ts:
Use a global cache to prevent creating multiple connections in
Next.js hot-reload environment.

  import mongoose from 'mongoose'

  const MONGODB_URI = process.env.MONGODB_URI!

  if (!MONGODB_URI) {
    throw new Error('MONGODB_URI environment variable is not defined')
  }

  interface GlobalMongoose {
    conn: typeof mongoose | null
    promise: Promise<typeof mongoose> | null
  }

  declare global {
    var _mongoose: GlobalMongoose | undefined
  }

  const cached: GlobalMongoose = global._mongoose ?? { conn: null, promise: null }
  global._mongoose = cached

  export async function connectToDatabase() {
    if (cached.conn) return cached.conn

    if (!cached.promise) {
      cached.promise = mongoose.connect(MONGODB_URI, {
        bufferCommands: false,
        maxPoolSize: 10,
        serverSelectionTimeoutMS: 5000,
        socketTimeoutMS: 45000,
      })
    }

    try {
      cached.conn = await cached.promise
    } catch (e) {
      cached.promise = null
      throw e
    }

    return cached.conn
  }

RULE 2: Every Mongoose model file MUST use this exact pattern:

  const ModelName = mongoose.models.ModelName ||
    mongoose.model<IModelDocument>('ModelName', ModelNameSchema)
  export default ModelName

  Never use just: mongoose.model('ModelName', schema)
  This causes "Cannot overwrite model once compiled" errors in Next.js.

RULE 3: Every API route that touches MongoDB must:
  - Call await connectToDatabase() at the start
  - Wrap the entire operation in try/catch
  - Return proper HTTP status codes

RULE 4: Never pass raw Mongoose documents from Server Components to
Client Components. Always serialize first:
  const data = await Model.find({}).lean()
  return JSON.parse(JSON.stringify(data))
  The .lean() method returns plain JS objects. JSON stringify/parse
  removes any remaining Mongoose-specific properties.

RULE 5: When creating documents, always use try/catch and check for
MongoDB error code 11000 (duplicate key) separately:

  try {
    await Model.create(data)
  } catch (err: unknown) {
    const mongoError = err as { code?: number; message?: string }
    if (mongoError.code === 11000) {
      return NextResponse.json({ error: 'Duplicate entry' }, { status: 409 })
    }
    throw err
  }

    } catch (err) {
      console.error('Failed to delete profile photo:', err)
    }
  }

  await Photo.deleteMany({ uploadedBy: id })
  await Vote.deleteMany({ voterId: id })
  await Notification.deleteMany({ userId: id })

ABSOLUTE RULES — VIOLATIONS WILL BREAK THE APP
════════════════════════════════════════════════════════════

1. Every file that uses useState, useEffect, useRef, Framer Motion,
   React Three Fiber, useSession, useRouter, usePathname,

   const Model = mongoose.models.Name || mongoose.model('Name', schema)

3. NEVER import cloudinary (the npm package) in any "use client" file.
   It is a Node.js package and will crash the browser build.

4. NEVER import mongoose or any MongoDB model in any "use client" file.
   Always use fetch() to call your own API routes from client components.

5. ALWAYS serialize MongoDB data before passing to client components:
   JSON.parse(JSON.stringify(data))

6. The /api/cloudinary/sign route DOES NOT EXIST in this app.
   All uploads go through /api/upload using server-side Cloudinary SDK.

7. Admin login is ONLY at /bash-mubc-bash/login.
   The public /signin page blocks admin accounts.
   No link to the admin login exists anywhere on the public site.

8. The cascade delete must delete Cloudinary images BEFORE deleting
   MongoDB documents. Never delete the database record without trying
   to clean up Cloudinary first.

9. One photo per user per contest week. Check this with an explicit
   database query in the API before creating the document. Do not
   rely on uniqueness alone since the photo type must also match.

10. All API routes that modify data must create an AdminLog entry
    if they are admin routes.

11. The pnpm run build must complete with zero TypeScript errors
    and zero Next.js build errors.

12. Never hardcode any credentials, API keys, or secrets in source
    code. All sensitive values must come from process.env.

13. All error responses from API routes must be JSON with an
    'error' string field and the correct HTTP status code.

14. Mongoose connection uses global cache. Never open multiple
    connections. The connectToDatabase() function in lib/mongodb.ts
    is the ONLY place mongoose.connect() is called.

15. In seed.ts, define schemas inline — never import from src/models
    because @/ path aliases don't resolve in tsx script context.
